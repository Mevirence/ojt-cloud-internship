From the start, a difference in UI is immediately noticeable across the three platforms. Azure, developed by Microsoft, keeps a traditional dashboard-style interface — familiar and straightforward, clearly designed with enterprise IT teams in mind. GCP, developed by Google, leans more toward a developer-oriented experience, with a denser and more detailed layout that surfaces more technical information upfront. AWS, developed by Amazon, sits somewhere in between — it resembles a traditional dashboard but presents more detail and configuration options than Azure. One standout observation is that Azure surfaces a direct GitHub integration prominently, making it easy to import and deploy code from a repository — something AWS and GCP don't offer as visibly out of the box. As cloud providers, none of these platforms require you to own any hardware; instead, you rent compute, storage, and networking on demand and only pay for what you use. 

From the setup alone, the three platforms already differ in what they offer. AWS provides $100 in free credits valid for up to 6 months, with an opportunity to earn an additional $100 by completing a set of onboarding tasks — each worth $20 in extra credits: using a foundation model in the Amazon Bedrock playground, creating a web app using AWS Lambda, creating an Aurora or RDS database, setting up a cost budget using AWS Budgets, and launching an EC2 instance. GCP offers $300 in free credits valid for 90 days, while Azure offers $200 valid for 30 days — neither of which provide opportunities to earn additional credits the way AWS does. On the identity and access side, creating users through AWS IAM, Azure Entra ID, and GCP Cloud IAM each felt noticeably different. AWS IAM felt the most feature-rich and intuitive to navigate, with more granular options surfaced clearly during setup. Personally, AWS is my preferred platform going forward — it offers the longest free usage window, the most familiar interface, and the most direct relevance to the AWS Cloud Practitioner certification I am working toward. That said, I intend to continue exploring Azure and GCP, as all three are widely used in the industry and understanding their differences is valuable in its own right.
